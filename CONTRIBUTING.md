# Contribution Guidelines

Thank you for your interest in contributing to Awesome AI Testing! This list aims to be a high-quality resource for testing AI and machine learning systems.

## Adding a New Entry

### Quality Standards

Before submitting, please ensure the tool/resource meets these criteria:

- **Actively maintained** - The project should have recent commits or updates (within the last year for open source projects)
- **Relevant to AI testing** - Must be related to testing, evaluating, or monitoring AI/ML systems
- **Not promotional spam** - Focus on genuine value to the community, not marketing
- **Stable and usable** - The tool should be ready for use, not just a concept or early prototype
- **Documented** - Should have clear documentation on how to use it

### Entry Format

Each entry should follow this format:

```markdown
- [Name](https://link.to/project) - Brief description ending with a period.
```

**Guidelines:**
- Keep descriptions concise (under 100 characters when possible)
- Start with a capital letter
- End with a period
- Avoid superlatives like "best", "fastest", "most popular"
- Don't include promotional language
- Use the official project name

### Pull Request Process

1. **Fork** this repository
2. **Add your entry** in the appropriate section, maintaining alphabetical order within each category
3. **One entry per PR** - This makes review easier
4. **Write a clear PR description** explaining:
   - What the tool/resource does
   - Why it belongs in this list
   - Which category it fits
5. **Check your links** - Ensure all URLs are working and use HTTPS where available

### Proposing New Categories

If your submission doesn't fit existing categories:

1. Open an issue first to discuss the new category
2. Explain why the category is needed
3. Provide at least 3 quality entries that would belong in it

## Updating Existing Entries

If a project has:
- Changed names or URLs
- Been deprecated or abandoned
- Significantly changed scope

Please submit a PR with the update and explain the change.

## Removing Entries

Entries may be removed if they:
- Are no longer maintained (2+ years without updates for open source)
- Have become unavailable
- No longer meet quality standards
- Have pivoted away from AI testing

## Code of Conduct

- Be respectful in discussions and reviews
- Accept constructive feedback gracefully
- Focus on what's best for the community

## Questions?

Open an issue if you have questions about whether something belongs on this list.
